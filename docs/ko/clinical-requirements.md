# 수의 임상 요구사항

*English: [`docs/en/clinical-requirements.md`](../en/clinical-requirements.md)*

현재의 "신호 추출 프로토타입"과 "임상에서 실제로 유용한 시스템" 사이의 공백을 임상 가치 순으로 정리.

> ⚠️ **데이터셋 특이사항**: 개들이 **단두종(불독·프렌치불독·시츄·퍼그)** 이고 배경에
> **마취기(Dräger/Fabius)** 가 보임 — 즉 *마취/회복 중 단두종*. 호흡·기도 baseline이 비정상이고
> 마취 위험이 높아 **고정 "개 정상범위"는 위험**.

## P0 — 최우선

1. **임상 검증.** 정답이 거친 모니터 OCR HR뿐. **동기화 gold standard**(ECG·펄스옥시미터·
   capnography·연속 체온)와의 일치도 검증 필요: Bland–Altman, 상태별 민감도/특이도,
   **종·품종·털색·체구·연령 층화**. 어두운/긴 털, 색소 침착 점막에서의 성능 저하 정량화.
2. **종/품종/개체별 baseline** (고정 임계값 대체). 정상치는 고양이≠개, 자견≠노령, 토이≠대형,
   그레이하운드=서맥 정상, 단두종=노력성 baseline. **개체 baseline 대비 편차**로 경보.
   → `petvitals/core/baselines.py` + `reports/patient_profiles/<stem>.json` 구현.
3. **호흡의 "질"(rate만으론 부족)**: 노력성·패턴·곤란(dyspnea)·복식/역설 호흡·무호흡,
   **수면 호흡수(SRR — 심장병 핵심)**, 단두종 stertor/stridor. → `petvitals/analyzers/respeffort.py`.

## P1 — 임상 기본

| 영역 | 필요 | 카메라 | 상태 |
|------|------|:---:|------|
| 관류 — 점막색(창백·청색증·황달·충혈) | 잇몸/혀/결막 색 | ★★ | ✅ `mucous.py` |
| 관류 — CRT, 혈압(NIBP/PTT) | 모세혈관 재충전, 혈압 대용 | ★ | 🔜 |
| 부정맥/리듬 | VPC·AF·AV block (rate 아님). **개의 RSA는 정상** → HRV 해석 종특이적 | ECG 필요 | 🔜 |
| 통증 | 검증 척도(개 Glasgow CMPS, 고양이 Grimace) — 표정+자세+행동 | ★★ | 🔜 |
| 의식/신경 | mentation(BAR/QAR/둔화/혼미/혼수), 발작, 머리기울임, 진전, 운동실조 | ★★ | 일부(pose) |
| 체중/수분 | 체중 추세(케이지 저울), 부종, 탈수 | 저울 ★ | 🔜 |

## P1–P2 — "모니터링" 시스템 요건

- **맥락 인식**: 마취/진정 상태·약물·ET튜브·IV라인·넥카라·붕대·담요의 ROI 가림 — 모르면 수치 무의미.
- **핸들러/장비 분리 + 개체 ID**; 다개체 케이지.
- **연속 추세 + 스마트 알람**: 30초 클립이 아니라 시간단위 + baseline 편차 경보 + 알람피로 관리 + 야간 IR.
- **기록/규제**: EMR 연동, 감사 추적, "의료기기 아님" 지위, 직원 에스컬레이션.

## 비접촉이 특히 강한 영역 (전략)

- **무자극 측정(fear-free)** — 특히 **고양이**(붙잡으면 흰가운 빈맥). 최대 차별점 → 고양이 확장 권장.
- **재택 수면 호흡수(SRR)** — 심장병 환자의 검증된 활용.
- **야간 무인 ICU** 감시.
