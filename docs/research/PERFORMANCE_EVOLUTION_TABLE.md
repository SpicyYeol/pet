# PET rPPG — 성능 진화 비교표 (전체 데이터 기준)

**목적**: 발표에서 "우리가 어떤 기술을 추가할 때마다 전체 성능이 어떻게 변화했는지"를 한눈에 보여주기 위한 표

**평가 기준**:
- 7개 usable 비디오 (1,3,4,5,6,7,8)
- Ground Truth: `video_labels_ocr.csv`의 `bpm_target`
- 지표: Mean Absolute BPM Error (전체 평균)

## 핵심 인사이트 (발표용 요약)

- **가장 큰 단일 개선**: A+B (생리적 신호 강화) 단계에서 고심박 비디오 성능이 극적으로 상승
- **Adaptive Selector의 가치**: "무조건 Multi"가 아니라 zone별로 선택하는 것이 전체 안정성을 가져옴 (특히 Video 6)
- **Dog-specific 모델의 효과**: v1에서 Video 7 단독 21.3 bpm이라는 가장 어려운 케이스에서 최고 성능 기록
- **중요한 교훈**: 단순히 데이터를 많이 넣는 것(v2)이 항상 좋은 것은 아님 → 고심박 데이터 비중이 중요

**추천 슬라이드 위치**: 
- "주요 인사이트" 또는 "결론 직전" 슬라이드

**발표자 대사 예시**:
> "지금까지 우리가 하나씩 추가해 온 기술들이 실제로 전체 데이터에서 어떤 영향을 주었는지 정리해 보았습니다. 
> 특히 A+B 단계와 Adaptive Selector, 그리고 dog-specific 모델이 고심박 비디오에서 결정적인 역할을 했다는 점을 확인할 수 있습니다.
> 마지막으로, 단순히 데이터를 많이 넣는 것이 항상 좋은 결과를 가져오는 것은 아니라는 중요한 교훈도 얻었습니다."
